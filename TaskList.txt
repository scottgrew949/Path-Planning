--------------------------------------------------------------------------------
  PROJECT OVERVIEW
--------------------------------------------------------------------------------

A ground-up C++ implementation of a robot navigation system that bridges
classical AI planning algorithms and modern reinforcement learning. The project
simulates the core software stack of an autonomous vehicle or mobile robot:
perceiving an environment, planning a path through it, and learning from
experience to improve over time.

The system is built entirely without external AI/ML libraries. Every algorithm,
data structure, and learning mechanism is implemented from first principles in
production-quality C++17.

WHAT IT DEMONSTRATES:
  - Mastery of classical graph search: A*, Dijkstra, BFS, Bidirectional A*,
    Theta* (any-angle planning with line-of-sight shortcuts)
  - Reinforcement learning from scratch: tabular Q-learning, Bellman equation,
    epsilon-greedy exploration, episode-based training
  - Deep RL: DQN, Double DQN, Dueling DQN via C++ environment + PyTorch bridge
  - Clean systems architecture: abstract interfaces, polymorphism, separation
    of concerns, zero external AI dependencies in the C++ core
  - Real-world relevance: every component maps directly to autonomous vehicle
    technology — occupancy grids, sensor fusion, motion planning, learned policy

SELF-DRIVING CAR ANALOG:
  Environment      = occupancy grid (what LiDAR produces)
  Classical algos  = global route planner (Google Maps layer)
  RL agent         = local motion policy (reacts to dynamic obstacles)
  DQN              = neural net replacing the lookup table for large state spaces
  Pi + Camera      = real-time perception pipeline feeding live obstacle data

HARDWARE EXTENSION (Raspberry Pi + Camera):
  A physical grid on a table with real obstacles detected by an overhead Pi
  camera feeds live data into the C++ environment. The RL agent navigates a
  grid that mirrors the physical world in real time — a miniature perception
  to planning to learning pipeline.

TARGET AUDIENCE: Robotics engineering, ML engineering, autonomous systems
                 research, computer science academia

--------------------------------------------------------------------------------
  SYSTEM ARCHITECTURE
--------------------------------------------------------------------------------

DIRECTORY STRUCTURE:

  core/
    Position.h/.cpp         — (x,y) coordinate type. Implements ==, !=, <.
                              PositionHash struct enables use as unordered_map key.
    Types.h/.cpp            — Project-wide enums and value structs:
                                CellType      — EMPTY, OBSTACLE, START, GOAL, PATH, VISITED
                                AlgorithmType — labels each algorithm for reporting
                                Action        — UP/DOWN/LEFT/RIGHT (int values 0-3)
                                PathResult    — bundles path + cost + time + nodes explored
                                StepResult    — bundles newPosition + reward + done flag

  environment/
    Cell.h/.cpp             — Single grid cell. Holds CellType and display character.
    Environment.h/.cpp      — The grid. Dual representation:
                                vector<vector<Cell>>   — full cell state for rendering
                                bitset<10000>          — O(1) obstacle lookup for planning
                              Key methods:
                                generateLabyrinth()    — recursive backtracking maze +
                                                         random wall removal for loops.
                                                         Guarantees a valid path exists.
                                getNeighbors()         — returns walkable adjacent cells
                                inBounds() / isObstacle() / isValid()
                                moveCost()             — edge weight (diagonal costs root 2)
                                reset()                — clears PATH/VISITED overlays

  planning/
    IPathfinder.h           — Abstract base class. Pure virtual interface:
                                findPath(), getName(), getType(), getNodesExplored()
                              Contains protected static reconstructPath() shared by
                              all algorithms. No duplicated path reconstruction logic.
    algorithms/
      AStar.h/.cpp          — Best-first search with heuristic (Manhattan distance).
                              Uses priority_queue + costFromStart unordered_map.
                              Finds optimal path, explores far fewer nodes than Dijkstra.
      Dijkstra.h/.cpp       — Uniform cost search. Explores in rings of equal cost.
                              Optimal but slower than A* — no directional bias.
      BFS.h/.cpp            — Unweighted shortest path. Queue-based, ignores edge costs.
      BidirectionalAStar.h/.cpp — Simultaneous forward (start to goal) and backward
                              (goal to start) A* searches that meet in the middle.
                              Explores ~3.5x fewer nodes than single-direction A*.
      ThetaStar.h/.cpp      — Any-angle planning. Checks line-of-sight from grandparent
                              directly to neighbor using Bresenham's algorithm.
                              If clear, skips current node — path not constrained to
                              grid edges. Returns smooth waypoint paths.

  rl/
    RLAgent.h/.cpp          — Abstract base class for all tabular RL agents.
                              Owns env_ ref, qTable_, all hyperparameters, PRNG.
                              Shared: train(), extractGreedyPath(), selectAction(),
                              updateQValue() (Bellman), decayEpsilon().
                              Pure virtual: runEpisode(). Mirrors IPathfinder pattern.
                              Constructor validates all hyperparameters — throws on bad values.
    RLEnvironment.h/.cpp    — Gym-style wrapper around Environment.
                              Exposes reset() and step(action) interface.
                              Reward design:
                                +100.0  goal reached
                                 -1.0   each valid step (encourages efficiency)
                                -10.0   wall or out-of-bounds hit
    QTable.h/.cpp           — State-action value store.
                              unordered_map<Position, array<double,4>, PositionHash>
                              One array of 4 doubles per visited cell (one per action).
                              getValue / setValue / getBestAction / getMaxValue
    QLearningAgent.h/.cpp   — Tabular Q-learning agent (inherits RLAgent).
                              Overrides runEpisode() — pure temporal difference learning.
                              One Bellman update per real environment step.
    DynaQAgent.h/.cpp       — Dyna-Q model-based RL agent (inherits RLAgent).
                              After each real step: updates world model + runs planningSteps_
                              additional Bellman updates replayed from the model.
                              model_: unordered_map (state,action) -> (reward, next_state).
                              modelKeys_: parallel vector for O(1) random sampling.
                              planningSteps_=0 degenerates to pure Q-learning.

  utils/
    ProbabilityUtils.h/.cpp — bayesUpdateSensor(), expectedValue(), entropy()

  visualization/
    Visualizer.h/.cpp       — displayGrid(), displayPath(), displayStats(),
                              displaySummaryTable()

  main.cpp                  — Full demonstration driver. All phases wired up.
  TaskList.txt              — This file.
  compile_commands.json     — clangd index for IDE tooling.

KEY DESIGN DECISIONS:

  grid_[x][y] layout         — outer vector is x (columns), inner is y (rows).
  IPathfinder interface       — all algorithms stored as vector<unique_ptr<IPathfinder>>.
                               Adding a new algorithm = new class, zero changes to main.
  reconstructPath in base     — shared protected static, no duplication across algos.
  bitset<10000> for obstacles — O(1) lookup called millions of times during search.
  PositionHash as struct      — avoids UB from std::hash specialisation.
  RLEnvironment by reference  — does not own the Environment. Caller controls lifetime.
  QTable lazy init            — only visited states stored, obstacle cells never appear.
  Hyperparameter validation   — constructor throws on invalid values. Silent bad
                               hyperparameters are impossible to debug after the fact.

STL CONTAINER MAP:

  vector<vector<Cell>>                    Environment grid (rendering)
  bitset<10000>                           Environment obstacle lookup
  unordered_set<Position,PositionHash>    BFS visited, Environment visited tracking
  unordered_map<Position,T,PositionHash>  A*, Dijkstra, BFS, ThetaStar cost/parent maps
  priority_queue + Comparator             A*, Dijkstra, ThetaStar open set
  queue<Position>                         BFS frontier
  array<double,4>                         QTable per-cell action values
  unordered_map<Position,array<double,4>> QTable full store
  vector<unique_ptr<IPathfinder>>         main.cpp algorithm collection
  vector<TrainingResult>                  QLearningAgent / DynaQAgent training history
  unordered_map<ModelKey,ModelValue,...>  DynaQAgent world model
  vector<ModelKey>                        DynaQAgent model keys (O(1) random sampling)

                   Output: ./pathplanning_classical

                   Output: ./pathplanning
